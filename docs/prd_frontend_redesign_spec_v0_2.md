# LKK POSM Web 应用前端重做 PRD 规格稿 v0.2

## 1. 修订目标

本轮修订用于指导 H5/Web 应用原型重做。现有原型存在两个核心问题：

1. 页面进入后已有大量预填内容，像演示假数据，不符合真实提交场景。
2. 表单右侧实时预览图让用户误以为系统会即时生成物料，和真实流程不一致。

新的产品原则：

- 首页只提供真实业务入口，不展示虚假的任务、统计、示例预览。
- 业务方先提交需求，系统异步生成，生成完成后才出现预览或结果链接。
- 用户可以查看自己提交的任务状态、溯源和历史记录。
- James 在应用内通过列表进入 Review，而不是在多维表格里完成主操作。
- 飞书多维表格只做台账、状态同步和业务/IT 管理视图，不作为 James 主 Review 工作台。

## 2. 角色与权限

| 角色 | 身份来源 | 核心能力 |
| --- | --- | --- |
| 需求方 | 飞书免登用户 | 单个提交、批量提交、查看自己的任务、查看自己的历史、下载已放行结果 |
| James / Studio Review | 飞书用户 + 后台配置的 Review 权限 | 查看 Review 列表、查看任务详情、拒绝、标记人工修正、确认通过并允许取用 |
| IT / 管理员 | 后台配置 | 查看全量台账、排查同步/通知/权限异常、维护模板和多维表格配置 |
| 系统 | 应用后端/定时任务 | 生成任务、上传云盘、写入 Base、发送机器人通知、记录溯源事件 |

权限规则：

- 需求方只能看到自己作为 `需求方` 的任务。
- James 可以看到分配给自己的 Review 任务；如果配置为 Studio 管理员，可查看全部 Review 任务。
- IT 可以查看全部任务和异常，但不代替 James 做业务通过动作。
- 业务取用通知只发送给 `需求方`，默认等于提交人，不新增取用人字段。

## 3. 信息架构

Web 应用分为 7 个主要页面：

| 页面 | 路径建议 | 目标用户 | 说明 |
| --- | --- | --- | --- |
| 工作台首页 | `/` | 全部用户 | 只展示提交入口、我的任务入口、历史记录入口；James 额外看到 Review 入口 |
| 提交需求 | `/submit` | 需求方 | 支持单个提交和批量 Excel 提交 |
| 我的任务 | `/tasks` | 需求方 | 查看自己进行中的任务、异常任务和可操作任务 |
| 任务详情与溯源 | `/tasks/:taskId` | 需求方 / James / IT | 查看参数、状态、结果链接、溯源时间线 |
| 历史记录 | `/history` | 需求方 | 查看已结束任务，支持复用历史需求 |
| James Review 列表 | `/review` | James | Review 工作台列表 |
| James Review 详情 | `/review/:taskId` | James | 预览生成结果、查看云盘目录、执行通过/拒绝/人工修正动作 |

首页约束：

- 不展示 mock 统计数字。
- 不展示示例任务卡片。
- 不展示实时 POSM 预览。
- 如果没有任务，只展示真实空状态。
- 如果当前用户不是 James，不显示 Review 入口。

## 4. 提交需求

### 4.1 提交模式

提交需求有两个模式：

| 模式 | 入口 | 适用场景 | 交互 |
| --- | --- | --- | --- |
| 单个提交 | 表单 | 单条 POSM 需求 | 用户填写字段后提交，校验通过后创建任务 |
| 批量提交 | Excel 模板 | 多条 POSM 需求 | 用户下载模板，填写后上传，系统逐行校验，用户确认后批量创建任务 |

默认进入 `/submit` 时：

- 默认选中“单个提交”。
- 除当前飞书用户身份外，业务字段全部为空。
- 不显示生成预览图。
- 右侧只显示“提交后将异步生成”的任务状态占位，不展示假素材。

### 4.2 单个提交字段

| 字段 | 字段 key | 必填 | 组件 | 默认值 | 校验 | 错误提示 |
| --- | --- | --- | --- | --- | --- | --- |
| 需求方 | requester | 是 | 只读用户信息 | 当前飞书用户 | 必须能解析飞书 open_id | 无法获取当前飞书用户，请重新进入应用 |
| Campaign | campaign | 是 | 下拉/搜索选择 | 空 | 必填 | 请选择 Campaign |
| 品牌/产品 | brand_product | 是 | 下拉选择 | 空 | 必填 | 请选择品牌/产品 |
| POSM 类型 | posm_type | 是 | 下拉选择 | 空 | 必填 | 请选择 POSM 类型 |
| 模板 | template_id | 是 | 下拉选择 | 空 | 必填；受 POSM 类型过滤 | 请选择模板 |
| 宽度 mm | width_mm | 是 | 数字输入 | 空 | 大于 0；符合模板上下限 | 请输入有效宽度 |
| 高度 mm | height_mm | 是 | 数字输入 | 空 | 大于 0；符合模板上下限 | 请输入有效高度 |
| 大区/城市/市场 | region_market | 是 | 下拉/级联选择 | 空 | 必填 | 请选择大区/城市/市场 |
| 语言 | language | 是 | 下拉选择 | 空 | 枚举值有效 | 请选择语言 |
| 数量 | quantity | 否 | 数字输入 | 空 | 如填写，必须为正整数 | 数量必须为正整数 |
| 交付用途 | delivery_purpose | 是 | 下拉选择 | 空 | 必填 | 请选择交付用途 |
| 期望完成日期 | deadline | 否 | 日期选择 | 空 | 不早于当天 | 期望完成日期不能早于今天 |
| 参考附件 | reference_files | 否 | 文件上传 | 空 | 单文件不超过配置上限；格式受限 | 附件格式或大小不符合要求 |
| 备注 | remark | 否 | 多行输入 | 空 | 不超过 500 字 | 备注不能超过 500 字 |

语言枚举建议：

- 简体中文
- 繁体中文
- 英文
- 中英双语

交付用途枚举建议：

- 门店陈列
- 活动物料
- KA 陈列
- 新品铺市
- 其他

单个提交规则：

- 前端只做缺失、格式、尺寸上下限和比例偏差等即时校验。
- 校验失败只在前端提示，不创建任务、不写入 Base、不上传云盘、不发通知。
- 校验通过后，调用创建任务接口，生成 `task_id`。
- 创建成功后跳转任务详情页。
- 创建失败时停留在当前表单，并展示失败原因。

### 4.3 批量提交

批量提交由 4 步组成：

1. 下载 Excel 模板。
2. 上传已填写 Excel。
3. 系统逐行校验并展示校验结果。
4. 用户确认后，系统只为校验通过的行创建任务。

上传规则：

| 规则 | 建议 |
| --- | --- |
| 文件格式 | `.xlsx` |
| 文件大小 | 不超过 10MB |
| 单次最大行数 | 500 行 |
| 空行处理 | 忽略完全空白行 |
| 表头校验 | 必须匹配系统模板 |
| 重复行处理 | 同一文件内关键字段完全一致时标记为重复，由用户确认是否继续 |
| 部分失败 | 支持部分成功；失败行不创建任务 |

Excel 模板列：

| 列名 | 必填 | 类型 | 校验 |
| --- | --- | --- | --- |
| Campaign | 是 | 文本/枚举 | 必须能匹配系统 Campaign |
| 品牌/产品 | 是 | 文本/枚举 | 必须能匹配系统配置 |
| POSM 类型 | 是 | 文本/枚举 | 必须为有效 POSM 类型 |
| 模板 | 是 | 文本/枚举 | 必须匹配 POSM 类型下的模板 |
| 宽度 mm | 是 | 数字 | 大于 0，符合模板上下限 |
| 高度 mm | 是 | 数字 | 大于 0，符合模板上下限 |
| 大区/城市/市场 | 是 | 文本/枚举 | 必须能匹配系统配置 |
| 语言 | 是 | 文本/枚举 | 必须为有效语言 |
| 数量 | 否 | 数字 | 如填写，必须为正整数 |
| 交付用途 | 是 | 文本/枚举 | 必须为有效交付用途 |
| 期望完成日期 | 否 | 日期 | 不早于当天 |
| 备注 | 否 | 文本 | 不超过 500 字 |

批量校验错误码：

| 错误码 | 说明 | 前端展示 |
| --- | --- | --- |
| REQUIRED_MISSING | 必填字段缺失 | 第 N 行：请补充必填字段 |
| INVALID_ENUM | 枚举值无效 | 第 N 行：字段值不在可选范围内 |
| INVALID_NUMBER | 数字格式无效 | 第 N 行：请输入有效数字 |
| OUT_OF_RANGE | 尺寸或比例超出模板限制 | 第 N 行：尺寸不符合模板限制 |
| INVALID_DATE | 日期格式或范围无效 | 第 N 行：日期格式不正确或早于今天 |
| DUPLICATE_ROW | 文件内重复需求 | 第 N 行：疑似重复需求 |
| TEMPLATE_NOT_FOUND | 模板不存在或不匹配 | 第 N 行：未找到匹配模板 |
| WRONG_TEMPLATE | Excel 表头不是系统模板 | 请下载最新模板后重新填写 |

批量提交成功后：

- 系统生成 `batch_id`。
- 每一条有效行生成独立 `task_id`。
- 用户进入批量结果页或我的任务页。
- 失败行支持下载错误报告。

## 5. 用户视角

### 5.1 我的任务

“我的任务”只展示当前用户仍在处理链路中的任务。

包含状态：

- 已提交
- 生成中
- 生成失败
- 待 Review
- 审核拒绝
- 人工修正中
- 已通过待通知
- 通知失败

列表字段：

| 字段 | 说明 |
| --- | --- |
| task_id | 任务编号 |
| submission_type | 单个提交 / 批量提交 |
| batch_id | 批量提交编号，单个提交为空 |
| Campaign | 业务活动 |
| POSM 类型 | 物料类型 |
| 尺寸 | 宽 x 高 mm |
| 大区/城市/市场 | 需求区域 |
| 提交时间 | 创建任务时间 |
| 当前状态 | 最新流程状态 |
| Review 人 | Studio Review |
| 最新进展 | 最新一条溯源事件摘要 |
| 操作 | 查看详情、下载结果、重新提交 |

操作规则：

- 未放行任务不能下载最终结果。
- `需求方是否可取用 = true` 且最终云盘链接非空时，才展示取用入口。
- 生成失败或审核拒绝的任务可以提供“复制为新需求”，但不会覆盖原任务。

### 5.2 任务详情与溯源

任务详情页包含：

| 模块 | 内容 |
| --- | --- |
| 基础信息 | task_id、batch_id、提交人、提交时间、Campaign、POSM 类型 |
| 需求参数 | 尺寸、模板、语言、区域、数量、用途、备注 |
| 生成结果 | 生成完成后展示 PNG 预览、云盘目录、最终文件链接 |
| Review 结果 | James 审核状态、审核意见、人工修正记录 |
| 通知记录 | 是否已通知需求方、通知时间、失败原因 |
| 溯源时间线 | 从提交到通知的完整事件 |

溯源事件结构：

| 字段 | 说明 |

| source | 事件来源：H5 / backend / scheduler / Base / Drive / bot |
| visibility | 对需求方可见 / 仅 James 与 IT 可见 |

事件类型枚举：

- SUBMITTED
- VALIDATION_FAILED
- BATCH_VALIDATED
- TASK_CREATED
- GENERATION_STARTED
- GENERATION_SUCCEEDED
- GENERATION_FAILED
- DRIVE_UPLOADED
- BASE_SYNCED
- REVIEW_REQUESTED
- REVIEW_REJECTED
- MANUAL_FIX_STARTED
- MANUAL_FIX_CONFIRMED
- RELEASE_APPROVED
- NOTIFICATION_SENT
- NOTIFICATION_FAILED

### 5.3 历史记录

“历史记录”展示当前用户已结束的任务。

包含状态：

- 已通知取用
- 已取消
- 已拒绝归档

过滤条件：

- 时间范围
- Campaign
- POSM 类型
- 状态
- 单个提交 / 批量提交
- 大区/城市/市场

历史复用规则：

- 可以复制：Campaign、品牌/产品、POSM 类型、模板、语言、大区/城市/市场、交付用途、备注。
- 必须重新确认：宽度、高度、期望完成日期、参考附件。
- 不复制：task_id、batch_id、生成结果链接、Review 状态、通知状态、溯源事件。

## 6. James Review

### 6.1 Review 列表

James 进入 `/review` 后看到 Review 工作台列表。

列表分组：

| Tab | 说明 |
| --- | --- |
| 待 Review | 生成完成，等待 James 审核 |
| 人工修正中 | James 已判断需要人工处理 |
| 已通过/已通知 | 已放行或已通知需求方 |
| 已拒绝 | 已明确拒绝且未进入人工修正 |
| 异常/超时 | 生成、上传、通知、权限、超时等异常 |

列表字段：

| 字段 | 说明 |
| --- | --- |
| task_id | 任务编号 |
| 需求方 | 提交人/业务取用通知对象 |
| Campaign | 业务活动 |
| POSM 类型 | 物料类型 |
| 尺寸 | 宽 x 高 mm |
| 大区/城市/市场 | 需求区域 |
| 生成完成时间 | 用于判断等待时长 |
| 等待时长 | 从进入待 Review 起计算 |
| 当前状态 | 流程状态 |
| 云盘目录 | 最终文件或目录链接 |
| 最新备注 | 最新 Review 或异常说明 |

筛选能力：

- 状态
- Campaign
- 需求方
- POSM 类型
- 提交时间
- 等待时长
- 是否异常

列表操作约束：

- 列表页只允许进入详情，不直接做放行动作。
- 放行、拒绝、人工修正必须在详情页完成，避免误操作。

### 6.2 Review 详情

Review 详情页展示：

- PNG 预览图：仅生成完成后展示，不做实时生成。
- 飞书云盘目录：展示原图、SVG、交付包或文件夹链接。
- 需求参数：Campaign、POSM 类型、模板、宽高、区域、语言、用途、备注。
- 需求方信息：需求方、提交时间、批量编号。
- 溯源事件：提交、生成、上传、Base 同步、通知等。
- Review 操作区：通过、拒绝、人工修正、确认修正后放行。

Review 动作规则：

| 当前状态 | James 动作 | 前置条件 | 下一状态 | 是否通知需求方 |
| --- | --- | --- | --- | --- |
| 待 Review | 通过并允许取用 | 云盘最终链接非空 | 已通过，随后发送通知 | 是 |
| 待 Review | 拒绝 | 必填拒绝原因 | 审核拒绝 | 否 |
| 待 Review | 标记人工修正中 | 必填原因 | 人工修正中 | 否 |
| 人工修正中 | 确认修正后放行 | 云盘最终链接非空；可选择更新链接 | 已通过，随后发送通知 | 是 |
| 审核拒绝 | 重新进入人工修正 | James 或 IT 权限 | 人工修正中 | 否 |
| 通知失败 | 重试通知 | 需求方与云盘链接非空 | 已通知取用或通知失败 | 成功后是 |

关键产品定义：

- MVP 中不拆分“审核通过”和“允许取用”两个按钮。
- James 点击“通过并允许取用”即代表业务放行。
- 系统随后设置 `需求方是否可取用 = true`，并通知 `需求方`。
- 如果通知失败，任务应停留在可重试的异常状态，而不是回退 Review 结果。
- 系统不得通过云盘目录是否出现新文件来判断 James 是否修正完成。
- James 人工修正完成后，必须回到应用页面点击“确认修正后放行”。

## 7. 状态机

| 状态 | 进入条件 | 用户可见 | James 可见 | 可执行动作 | 下一状态 |
| --- | --- | --- | --- | --- | --- |
| 草稿 | 用户填写中 | 是 | 否 | 提交 / 取消 | 已提交 |
| 已提交 | 表单或 Excel 行创建成功 | 是 | 否 | 查看详情 | 生成中 |
| 生成中 | 后端开始生成 | 是 | 否 | 查看详情 | 待 Review / 生成失败 |
| 生成失败 | 生成服务失败 | 是 | IT 可见 | 复制为新需求 / IT 重试 | 生成中 / 归档 |
| 待 Review | 生成成功、云盘上传、Base 同步完成 | 是 | 是 | James Review | 已通过 / 审核拒绝 / 人工修正中 |
| 审核拒绝 | James 拒绝 | 是 | 是 | 复制为新需求 / 重新进入人工修正 | 人工修正中 / 归档 |
| 人工修正中 | James 标记需要人工处理 | 是 | 是 | James 确认修正后放行 | 已通过 |
| 已通过 | James 显式放行 | 是 | 是 | 系统发送通知 | 已通知取用 / 通知失败 |
| 通知失败 | 机器人通知失败 | 是 | 是 | IT 或系统重试 | 已通知取用 / 通知失败 |
| 已通知取用 | 通知需求方成功 | 是 | 是 | 下载 / 查看历史 | 归档 |

最终取用门禁：

- `需求方是否可取用 = true`
- `需求方` 非空
- `POSM SVG` 或最终云盘目录链接非空
- 通知成功后状态进入 `已通知取用`

## 8. 飞书多维表格定位

Base 是轻量台账、状态同步面和业务/IT 管理视图，不是文件仓库，也不是 James 的主 Review 页面。

Base 字段使用原则：

| 字段 | 用途 |
| --- | --- |
| 需求方 | 最终通知对象，默认等于提交人 |
| 是否确认开始生成 | 任务触发门禁 |
| POSM 类型 | 生成入参和台账字段 |
| Campaign 类型 | 生成入参和台账字段 |
| POSM 高（mm）（限制最大/小） | 生成入参和校验依据 |
| POSM 宽（mm）（限制） | 生成入参和校验依据 |
| 比例（自动算+限制） | 校验依据 |
| POSM PNG | 预览图链接 |
| POSM SVG | 最终云盘目录或文件链接 |
| POSM 生成状态 | 流程状态展示 |
| Studio Review | James / Review 负责人 |
| 需求方是否可取用 | 最终放行和通知门禁 |

Base 承载量策略：

- Base 不存大文件，只存链接、状态、轻量文本和必要索引。
- 文件本体存飞书云盘。
- 历史数据后续按 Campaign、月份或年份归档/拆表。
- 应用页面查询应优先走业务后端索引，避免前端直接依赖大 Base 全量扫描。

## 9. 通知规则

| 触发点 | 收件人 | 渠道 | 内容 | 失败处理 |
| --- | --- | --- | --- | --- |
| 生成完成待 Review | Studio Review | 飞书机器人 | 任务数、Review 入口、等待时长 | 记录通知失败，进入异常列表 |
| James 拒绝 | 默认不通知需求方 | 无 | 无 | 状态在任务详情可见 |
| James 标记人工修正 | 默认不通知需求方 | 无 | 无 | 状态在任务详情可见 |
| James 放行 | 需求方 | 飞书机器人 | task_id、Campaign、取用链接、任务详情入口 | 失败进入通知失败，可重试 |
| 通知重试成功 | 需求方 | 飞书机器人 | 同放行通知 | 更新为已通知取用 |

3 小时收割逻辑：

- 定时扫描进入 `待 Review` 且未提醒或达到再次提醒间隔的任务。
- 只提醒 `Studio Review`。
- 不通知 `需求方`。

## 10. 异常与边界

| 场景 | 页面提示 | 所有人 | 下一步 |
| --- | --- | --- | --- |
| 单个提交校验失败 | 请修正标红字段后再提交 | 需求方 | 停留在表单 |
| Excel 空文件 | 未识别到有效需求行 | 需求方 | 重新上传 |
| Excel 模板错误 | 请下载最新模板后重新填写 | 需求方 | 重新下载模板 |
| Excel 部分行失败 | 部分需求未通过校验，可下载错误报告 | 需求方 | 修正后重传，或只提交通过行 |
| 生成失败 | 系统生成失败，已记录异常 | IT | 重试或排查 |
| 云盘上传失败 | 文件上传失败，暂不可 Review | IT | 重试上传 |
| Base 同步失败 | 台账同步失败，不影响文件安全 | IT | 重试同步 |
| Review 拒绝 | 当前结果未通过 Review | 需求方 / James | 复制为新需求或人工修正 |
| 人工修正超时 | 人工修正中超过配置时长 | James / IT | 催办或关闭 |
| 云盘链接过期或无权限 | 当前链接暂不可访问 | IT | 修复权限或重新生成链接 |
| 通知失败 | 已放行，但通知发送失败 | IT | 重试通知 |

## 11. 前端硬约束

为避免页面继续显得“fake”，开发时必须遵守：

- 首页不放演示任务卡片。
- 表单业务字段默认全部为空。
- 不展示实时生成预览。
- 只有任务生成完成后才展示 PNG 预览图。
- 不展示无法被真实数据支撑的统计数字。
- 所有任务列表数据必须来自 mock API 或真实 API，不写死在 UI 组件里。
- mock 数据必须集中在 adapter/mock 层，页面组件不能直接散落假数据。
- 空状态必须是真空状态，不用示例素材填充。
- James 放行前，不出现“已通知需求方”状态。

## 12. 验收标准

### 12.1 首页

- Given 当前用户没有任何任务，When 进入首页，Then 页面只展示提交入口、我的任务入口、历史入口和真实空状态。
- Given 当前用户不是 James，When 进入首页，Then 不展示 Review 入口。
- Given 当前用户是 James，When 进入首页，Then 展示 James Review 入口。

### 12.2 单个提交

- Given 用户首次进入提交页，When 页面加载完成，Then 除当前用户身份外所有业务字段为空。
- Given 必填字段缺失，When 用户点击提交，Then 页面提示缺失字段且不创建任务。
- Given 尺寸超出模板限制，When 用户点击提交，Then 前端提示尺寸错误且不写 Base、不上传云盘、不发通知。
- Given 表单校验通过，When 用户点击提交，Then 创建任务并跳转任务详情页。

### 12.3 批量提交

- Given 用户上传错误模板，When 系统校验，Then 提示下载最新模板。
- Given Excel 中存在错误行，When 校验完成，Then 展示逐行错误并允许下载错误报告。
- Given Excel 中部分行通过，When 用户确认提交，Then 只为通过行创建任务。
- Given 批量提交成功，When 用户进入我的任务，Then 可以看到每条有效行对应的任务。

### 12.4 用户任务与历史

- Given 用户进入我的任务，When 查询任务列表，Then 只看到自己提交且未结束的任务。
- Given 任务已通知取用，When 用户进入历史记录，Then 可以查看该任务并打开最终云盘链接。
- Given 用户点击复用历史，When 进入提交页，Then 仅复制允许复用的业务字段，不复制结果链接和状态。

### 12.5 James Review

- Given 任务处于待 Review，When James 进入 Review 列表，Then 可以在待 Review Tab 看到该任务。
- Given James 点击拒绝，When 未填写原因，Then 系统阻止提交并提示必须填写原因。
- Given James 拒绝任务，When 状态更新完成，Then 不通知需求方。
- Given James 标记人工修正中，When 云盘目录有新文件，Then 系统不自动通知需求方。
- Given James 点击确认修正后放行，When 最终云盘链接非空，Then 系统设置可取用并通知需求方。

### 12.6 通知与异常

- Given James 已放行但机器人通知失败，When 用户查看任务详情，Then 状态显示通知失败且保留取用链接。
- Given IT 重试通知成功，When 状态刷新，Then 任务进入已通知取用。
- Given Base 同步失败，When IT 查看异常列表，Then 可以看到同步失败原因和重试入口。

## 13. 本轮开发范围

本轮建议只做前端原型和 mock API：

- Web 横向工作台布局。
- 首页真实入口。
- 单个提交表单。
- 批量提交上传与校验结果 mock。
- 我的任务列表。
- 历史记录列表。
- 任务详情与溯源。
- James Review 列表。
- James Review 详情与状态流转。

不在本轮实现：

- 真实飞书免登。
- 真实飞书 Base 写入。
- 真实飞书云盘上传。
- 真实飞书机器人通知。
- 真实 Excel 服务端解析。
- 真实生成服务。

这些能力应通过 `api.ts`、`feishu.ts`、`baseAdapter.ts`、`driveAdapter.ts`、`botAdapter.ts` 等 adapter/mock 接口隔离，便于后续替换。
